Emoji to Image Replacement System

This system replaces hardcoded emoji characters with actual images from our image library, providing a consistent visual experience throughout the application.

Features

  • Automatic Replacement: Emojis with mappings are automatically replaced with our custom images
  • Fallback Support: Shows original emoji if image fails to load
  • Configurable Sizing: Different emoji sizes for different contexts
  • Accessibility: Proper alt text and title attributes
  • Slot Support: Can be used as wrapper component with slots
  • 30+ Emoji Mappings: Comprehensive mapping for business, creative, and UI emojis

Usage Examples

1. EmojiText Component (Recommended)

Replace emojis in text strings with a single component:

<template>
  <EmojiText 
    :text="'Made with ❤️ in Amsterdam'" 
    :emoji-size="16" 
  />
</template>

<script setup>
import EmojiText from '@/components/ui/EmojiText.vue'
</script>

2. EmojiReplacer Component

Individual emoji replacement with full control:

<template>
  <EmojiReplacer 
    emoji="❤️" 
    :size="24" 
    title="Love this app"
  />
</template>

3. useEmojiReplacer Composable

Programmatic emoji replacement for complex scenarios:

import { useEmojiReplacer } from '@/composables/useEmojiReplacer'

const { createEmojiText } = useEmojiReplacer()

const emojiNodes = createEmojiText('❤️ Love you!', {
  size: 20,
  emojiTitle: 'Heart'
})

Benefits

  1. Consistency: All emojis use the same visual style and quality
  2. Performance: Images are loaded from our CDN, no external dependencies
  3. Accessibility: Proper alt text and semantic markup
  4. Maintainability: Centralized emoji mapping for easy updates
  5. Fallback Safety: Original emoji shown if image loading fails
  6. Size Flexibility: Different emoji sizes for different UI contexts

Migration Guide

To replace hardcoded emojis in existing components:

  1. Find emojis using the mapping: find . -name "*.vue" -exec grep -l "❤️" {} \;
  2. Replace with EmojiText or EmojiReplacer components
  3. Import the required components and composables
  4. Test both image loading and fallback scenarios

Best Practices

  • Use EmojiText for text strings containing emojis
  • Use EmojiReplacer for individual emoji display
  • Set appropriate emojiSize for context (16-48px)
  • Provide descriptive title attributes for accessibility
  • Test fallback behavior with network issues
  • Use semantic emojis that match the intended meaning
